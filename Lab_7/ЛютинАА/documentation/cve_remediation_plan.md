## Условия эскалации

Принятые риски автоматически переходят в P0 при выполнении любого из условий:

| Условие | CVE | Действие |
|---------|-----|---------|
| Debian выпустил патч | Любой принятый High CVE | Обновить базовый образ `python:3.14-slim` в течение 48 часов |
| CVSS переоценён до ≥ 9.0 | Любой принятый High CVE | Срочное обновление образа + перезапуск production |
| Появился публичный exploit (PoC) | CVE-2026-4878, CVE-2026-29111 | Перейти на distroless или Alpine + пересборка C-расширений |
| Trivy обнаружил Critical CVE | Новый CRITICAL | Блокирует CI немедленно, требует fix до следующего деплоя |

## Процедура обновления базового образа

При появлении патчей для принятых CVE:

```bash
# 1. Обновить тег в Dockerfile (если используется фиксированный digest):
#    FROM python:3.14-slim → FROM python:3.14-slim@sha256:<новый_дайджест>

# 2. Запустить локальный Trivy scan:
docker build -t shiftcontrol-backend:test ./backend
trivy image shiftcontrol-backend:test --severity CRITICAL,HIGH

# 3. Убедиться что исправленные CVE исчезли из отчёта

# 4. Запушить изменение — CI автоматически прогонит trivy-image в pipeline
```
